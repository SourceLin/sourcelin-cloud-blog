# 升级指南

本指南用于在版本迭代时，帮助您完成平滑升级。若当前暂无明确升级路径，可按照下列通用步骤进行准备与验证。

## 适用范围
- 适用版本：v1.0.x（示例）
- 目标版本：以发布说明为准

## 升级前准备
1. 备份数据库与配置文件。
2. 记录当前运行的服务版本与依赖版本。
3. 确认本次升级的变更范围（接口、配置、数据库）。

## 数据库升级
1. 备份数据库（强烈建议）。
2. 执行版本对应的 SQL 变更脚本（如有）。
3. 校验关键表结构与索引是否正确。

## 配置更新
1. 对比新版配置与现有配置差异。
2. 合并必要配置项并记录变更。
3. 检查 Nacos 配置是否同步更新。

## 服务升级流程
1. 停止相关服务。
2. 更新服务包或镜像。
3. 按顺序启动服务（建议：网关与认证最后启动）。
4. 进行冒烟测试与回归验证。

## 验证清单
- 登录与鉴权正常
- 核心接口可用
- 前台页面渲染正常
- 后台管理功能正常
- 监控与日志正常

## 常见问题
- 若出现配置缺失，请对照新版配置文件补齐。
- 若接口报错，请确认数据库变更已执行。
- 若页面异常，请清理前端缓存并重新构建。
