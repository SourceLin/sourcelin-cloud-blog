# 支持说明

Sourcelin Blog 是持续演进中的开源项目。这里整理项目支持方式，方便使用者快速找到合适的沟通入口。

## 首选支持渠道

优先使用 Gitee Issue 反馈问题：

- 仓库地址：[https://gitee.com/my_lyq/sourcelin-cloud-blog](https://gitee.com/my_lyq/sourcelin-cloud-blog)
- Issues：[https://gitee.com/my_lyq/sourcelin-cloud-blog/issues](https://gitee.com/my_lyq/sourcelin-cloud-blog/issues)

适合通过 Issue 反馈的内容：

- 启动失败
- 页面报错
- 接口异常
- 文档问题
- 功能建议
- 二次开发疑问

## 免费支持范围

以下内容可以优先通过 Issue 或邮件沟通：

- 环境配置答疑
- 本地启动问题排查
- 常见部署问题说明
- 文档勘误
- 简单使用咨询

## 企业与定制支持

如果你有以下需求，可以邮件联系进一步沟通：

- 私有化部署
- 定制开发
- 业务二开评估
- 培训和项目讲解
- 安全加固和部署方案整理

联系邮箱：lyq_0201@126.com

## 如何更高效地获得帮助

无论是提 Issue 还是发邮件，建议尽量附带：

1. 使用场景
2. 部署方式
3. 复现步骤
4. 日志截图或报错堆栈
5. 当前分支和修改情况

这样能明显减少来回沟通成本。

## 支持项目的方式

如果项目对你有帮助，欢迎通过以下方式支持：

- 点一个 Star
- 提交 Issue 或 PR 帮助项目完善
- 分享你的部署案例或二开案例
- 把项目推荐给有需要的开发者

感谢支持。
