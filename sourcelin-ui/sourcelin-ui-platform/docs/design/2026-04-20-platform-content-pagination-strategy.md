# 前台内容流与索引页分页策略方案

## 1. 范围

本方案统一评估以下 6 个页面的列表加载与分页策略：

- `src/modules/say/pages/SayPage.vue`
- `src/modules/treehole/pages/TreeholePage.vue`
- `src/modules/hot/pages/HotPage.vue`
- `src/modules/article/pages/ArchivePage.vue`
- `src/modules/article/pages/TagPage.vue`
- `src/modules/article/pages/CategoryPage.vue`

目标不是把所有页面强行做成同一种分页，而是让：

1. 信息架构完整
2. 交互符合页面语义
3. UI 符合当前产品的 glass / panel / quiet feedback 设计语言
4. 状态模型可持续扩展

---

## 2. 总结论

这 6 个页面应分成 **4 类列表模式**，不要统一成一种分页模型。

### A. 动态流

- 说说页

推荐：**无限滚动主导 + 弱分页补充**

### B. 讨论流

- 树洞页

推荐：**显式分页主导**

### C. 榜单流

- 热门页

推荐：**手动渐进加载主导 + 状态分页补充**

### D. 索引页

- 归档页
- 标签页
- 分类页

推荐：

- 归档页：**时间分段加载，不用传统页码优先**
- 标签页 / 分类页：**显式分页主导**

---

## 3. 当前实现判断

## 3.1 说说页

现状：

- `IntersectionObserver` + `page/pageSize/hasMore`
- 自动追加下一页
- 无显式分页
- 底部仅有 `loadMoreSentinel`

判断：

- 作为动态流，保留无限滚动是合理的；
- 但当前只有“继续往下刷”的单向体验，没有“我现在在哪一页 / 已经加载多少 / 如何回到旧批次”的闭环；
- 结论：**方向合理，闭环不足。**

## 3.2 树洞页

现状：

- 主列表只请求 `page: 1, pageSize: 50`
- 主列表无分页器
- 评论区内部已有分页

判断：

- 页面本体是“内容 + 讨论”双层结构，不适合伪无限流；
- 主列表没有翻页时，旧树洞不可达；
- 评论有分页、主列表无分页，信息导航策略不一致；
- 结论：**当前不合理，应改成显式分页。**

## 3.3 热门页

现状：

- `useHotPage.ts` 维护 `page/pageSize/finished`
- 使用 `WaterfallList`
- 底部通过 `@load-more="loadMore"` 手动加载更多
- 不是无限滚动

判断：

- 热榜是“已精选 + 继续展开”的浏览模型；
- 手动加载更多符合榜单语义，比无限滚动更稳；
- 但目前缺少明确页次 / 已加载批次反馈；
- 结论：**当前基本合理，但应补轻量分页状态。**

## 3.4 归档页

现状：

- `loadArchives()` 直接请求 `page: 1, pageSize: 1000`
- 前端一次性分组成“年月归档”
- 无分页、无分段加载

判断：

- 归档页的核心语义是“时间索引”，不是普通卡片列表；
- 因此它不适合强行套普通页码分页；
- 但一次性拉 1000 条也不合理，后续数据增长会直接恶化首屏与内存；
- 结论：**不应做普通页码优先，而应做时间分段加载。**

## 3.5 分类页

现状：

- 分类列表浏览正常
- 切到某分类后，相关文章固定请求 `page: 1, pageSize: 12`
- 无下一页、无总数导航

判断：

- 这是标准“上下文索引页”而不是纯 feed；
- 用户在一个分类下通常有明确继续翻页预期；
- 只取 12 条而无分页，会把更多内容静默截断；
- 结论：**应改为显式分页主导。**

## 3.6 标签页

现状：

- 标签浏览区正常
- 切到某标签后，相关文章固定请求 `page: 1, pageSize: 12`
- 无下一页、无总量感知

判断：

- 与分类页同类，但标签语义更像“关键词索引”；
- 用户对结果集的“继续翻页”预期更强；
- 结论：**应改为显式分页主导。**

---

## 4. 结合产品设计语言的统一原则

本产品不是后台表格系统，分页设计不能走重型控件堆砌路线。

当前前台设计语言的关键特征是：

- Hero 先行
- glass / panel 分层
- 卡片主导
- quiet loading / quiet feedback
- `SSurfaceChip`、`SPagination`、`SSurfacePanel` 这一套轻控制语言

所以分页方案必须遵循：

1. 不在所有页面都上“重页码条”
2. 不让 feed 页突然变后台列表
3. 不让索引页只剩视觉，没有导航闭环
4. 交互层使用已有 UI 抽象，而不是页面自己发明一套 pager

---

## 5. Naive UI 可用能力

当前版本：

- `naive-ui@2.44.1`

## 5.1 可直接用

### `NPagination`

项目已通过 `SPagination` 封装接入，可用于：

- 树洞主列表分页
- 分类页相关文章分页
- 标签页相关文章分页
- 热门页的弱分页补充
- 说说页的 `simple` 弱导航补充

适合使用的能力：

- `simple`
- `page`
- `pageSize`
- `itemCount`
- `showQuickJumper`
- `prefix`
- `suffix`

### `NBackTop`

适合：

- 说说页
- 热门页
- 归档页

作用：

- 补充长页面回顶体验
- 不替代分页，但改善长内容流可操作性

### `NAnchor`

适合：

- 归档页的年份 / 月份索引跳转

不适合直接用于：

- 说说 / 树洞 feed 流

## 5.2 不存在或不建议依赖

### 页面级 `NInfiniteScroll`

当前版本没有现成可直接使用的 `NInfiniteScroll` 组件可接页面列表。

结论：

- 无限滚动仍需项目自己维护状态机
- Naive UI 不能替代：
  - `IntersectionObserver`
  - 已加载批次管理
  - URL / 历史恢复

### `NScrollbar` 做内层滚动分页

不推荐。

原因：

- 会破坏整页滚动体验
- 与当前 `PageShell + Section` 结构冲突
- 对移动端和评论展开交互不友好

## 6. 分页策略矩阵

| 页面 | 当前问题 | 推荐模式 | 是否用 `SPagination` | 是否保留无限滚动 |
|------|----------|----------|----------------------|------------------|
| 说说页 | 只有 append，无位置感 | 无限滚动主导 + 弱分页 | 可选，弱导航 | 是 |
| 树洞页 | 只拉第一页 50 条 | 显式分页主导 | 是，主控件 | 否 |
| 热门页 | 有加载更多，无页次反馈 | 手动渐进加载 + 弱分页 | 是，补充 | 否 |
| 归档页 | 一次性拉 1000 条 | 时间分段加载 | 不做传统页码优先 | 否 |
| 分类页 | 只取 12 条，无翻页 | 显式分页主导 | 是，主控件 | 否 |
| 标签页 | 只取 12 条，无翻页 | 显式分页主导 | 是，主控件 | 否 |

## 7. 各页面方案

## 7.1 说说页

### 推荐方案

采用：**无限滚动主导 + 弱分页补充**

### 交互结构

1. 保留当前 `IntersectionObserver` 自动加载
2. 在底部哨兵区域增加：
   - 已加载 `x / total`
   - 当前第 `n / totalPages` 页
3. 前 2~3 页自动加载，之后切为手动“继续查看更早动态”
4. 桌面端可选追加 `SPagination simple` 作为弱导航
5. 配合 `NBackTop` 或现有回顶能力

### UI 表现

- 状态条使用 `SSurfaceChip` / quiet 文案
- 不引入大面积页码控件
- 保持 timeline / feed 气质

### 理由

说说页用户目标是“沉浸式刷动态”，不是精准跳第 7 页；
但没有页次反馈会让体验漂浮，所以应补“弱分页”。

---

## 7.2 树洞页

### 推荐方案

采用：**显式分页主导**

### 交互结构

1. `useTreeholePage.ts` 增加：
   - `page`
   - `pageSize`
   - `total`
2. 页面主列表底部增加 `SPagination`
3. 翻页后自动滚到 `.messages-section` 顶部
4. 翻页时关闭当前展开评论，清理 `activeTreeholeId`
5. 评论区分页继续保持现状

### UI 表现

- 分页器放在 `messages-section` 底部
- 上方保留总数摘要
- 移动端使用 `simple`
- 桌面端使用标准模式 + 总数说明

### 理由

树洞是“匿名内容流 + 讨论列表”，更接近索引页，不适合伪无限流。

---

## 7.3 热门页

### 推荐方案

采用：**手动渐进加载主导 + 状态分页补充**

### 当前状态评价

热门页当前用 `WaterfallList + loadMore`，本质已经不是无限滚动，而是较合理的“查看更多榜单内容”。

### 建议补强

1. 保留当前“继续浏览榜单”手动加载
2. 增加弱分页状态：
   - 当前第 `n / totalPages` 页
   - 已加载 `x / total`
3. 当榜单分类切换时，明确回到第 1 页状态
4. 可选给底部补 `SPagination simple`，用于桌面端快速回前一页 / 后一页

### UI 表现

- 不建议改成自动无限滚动
- 维持榜单的“分批展开”感
- 状态文案可放进 `section-count` 或底部 quiet bar

### 理由

热榜不是时间流，而是“精选榜单”；手动加载比无限滚动更符合榜单心理模型。

---

## 7.4 归档页

### 推荐方案

采用：**时间分段加载，不以传统页码为第一优先**

### 当前问题

- 直接请求 `pageSize: 1000`
- 前端一次性组装归档
- 数据越多越不可持续

### 建议交互

1. 首屏只加载最近若干个月或一个合理批次
2. 页面顶部或侧边提供：
   - 年份快速跳转
   - 月份锚点导航
3. 底部使用“加载更早归档”逐段追加
4. 配合 `NAnchor` 或自定义年份索引
5. 配合 `NBackTop`

### UI 表现

- 归档页更像时间地图，不像普通列表分页
- 不建议直接做“1 2 3 4 5 页”
- 更适合：
  - 年份导航
  - 月份分段
  - 逐段加载更早内容

### 理由

归档的用户目标通常是按时间定位，而不是按页码定位。

---

## 7.5 分类页

### 推荐方案

采用：**显式分页主导**

### 当前问题

- 当前分类文章只请求 `pageSize: 12`
- 没有下一页
- 结果被静默截断

### 建议交互

1. `loadArticlesForCategory()` 改为接收 `page/pageSize`
2. 详情态底部加入 `SPagination`
3. 页码切换后滚回 `categorySwitchSectionRef`
4. 切换分类时重置页码到 1
5. 在上下文条中增加：
   - `共 x 篇`
   - `当前第 n 页`

### UI 表现

- 详情态底部使用 `SPagination`
- 移动端切 `simple`
- 顶部焦点条保留，不要破坏当前沉浸式切换结构

### 理由

分类页属于“明确主题下的文章索引”，显式分页最稳定。

---

## 7.6 标签页

### 推荐方案

采用：**显式分页主导**

### 当前问题

- 当前标签文章只请求 `pageSize: 12`
- 无翻页
- 标签天然具备长尾结果集特征

### 建议交互

1. `loadArticlesForTag()` 改为接收 `page/pageSize`
2. 详情态底部加入 `SPagination`
3. 切页滚回 `tagSwitchSectionRef`
4. 切换标签时页码归 1
5. 当前标签面板中增加：
   - `共 x 篇`
   - `当前第 n 页`

### UI 表现

- 与分类页保持同一分页语言
- 标签页因为关键词密度高，可在顶部保留当前标签摘要，但不要把翻页埋起来

### 理由

标签页本质是关键词搜索结果的主题视图，用户对分页预期非常明确。

---

## 8. 统一实施原则

### 8.1 页面级状态机要统一抽象，但交互不要统一外观

建议抽一个共享的页面列表分页 composable，例如：

- `usePagedFeed()`
- 或 `usePagedIndexView()`

统一管理：

- `page`
- `pageSize`
- `total`
- `totalPages`
- `loading`
- `finished`
- `loadPage`
- `loadMore`
- `reset`

但页面展示策略分开：

- 说说页：feed 模式
- 树洞 / 分类 / 标签：pager 模式
- 热门：progressive mode
- 归档：time-section mode

### 8.2 统一复用现有 UI 抽象层

优先使用：

- `SPagination`
- `SSurfaceChip`
- `SSurfacePanel`
- `SButton`

不要在页面层重新发明：

- 自制页码条
- 大量临时 chip 状态块
- 与现有 pagination shell 不一致的翻页按钮

---

## 9. 实施优先级

### P0

- 树洞主列表显式分页
- 分类页详情态显式分页
- 标签页详情态显式分页

### P1

- 说说页补弱分页状态条
- 热门页补页次 / 已加载状态

### P2

- 归档页改为时间分段加载 + 锚点导航
- 说说页支持恢复最近浏览批次
- 热门页可选补 `simple` 分页弱导航

---

## 10. 一句话结论

- **说说页**：保留无限滚动，但必须补上弱分页与状态闭环。
- **树洞页**：改成显式分页主导。
- **热门页**：维持手动渐进加载，只补状态分页反馈。
- **归档页**：不要做普通页码优先，应该做时间分段加载。
- **分类页 / 标签页**：必须接入显式分页，否则结果集会被静默截断。
- **Naive UI**：能很好承接分页 UI（`NPagination`、`NAnchor`、`NBackTop`），但不能替代页面自己的加载状态机。
