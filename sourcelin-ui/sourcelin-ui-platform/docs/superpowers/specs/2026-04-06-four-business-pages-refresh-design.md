# 四个业务页面混合型 Hero 重构设计

**日期**：2026-04-06
**范围**：`HotPage`、`AboutPage`、`NavigationPage`、`UserPage`
**说明页面**：首页、文章分类、文章标签、说说、树洞、友情链接

## 1. 目标

在不改动接口协议、路由结构和核心业务逻辑的前提下，重构四个页面的头部与内容区排版，让它们共享同一套站点视觉语言，同时保留明显的页面个性。

最终效果要求：

- 页面头部统一为“混合型 Hero”：左侧编辑感标题区，右侧上下文面板，底部统计卡或引导块
- 内容区统一为 workspace 章节化结构，而不是简单的卡片堆叠
- `热门 / 关于本站 / 个人中心` 在排版上明显个性化
- `网站导航` 更强调秩序、章节感和检索效率
- 允许新增前端可闭环的小功能，但不新增后端依赖

## 2. 统一骨架

### 2.1 Hero 骨架

四个页面统一使用以下视觉骨架：

- `PageShell` 负责页面宽度与外层节奏
- `FeatureCard` 作为 hero 主容器
- Hero 左侧统一包含：眉题、超大标题、导语、统计卡
- Hero 右侧统一包含：当前页上下文面板、快捷说明或状态摘要
- 移动端统一塌缩为单列，右侧面板下沉到标题区之后

### 2.2 内容区骨架

- 每个主模块都升级为 `section workspace`
- 每个模块统一具备：`section-head`、标题、副文案、数量 badge、局部操作区
- 模块之间使用一致的内外层级：`sourcelin-panel-soft`、`sourcelin-panel`、`sourcelin-panel-inset`
- 避免页面里出现只有标题没有模块头、只有卡片列表没有章节语义的平铺结构

## 3. 页面个性化策略

### 3.1 热门

定位为“榜单编辑台”。

- Hero 使用更强的排名感和频道感
- 顶部增加当前榜单说明、分类状态、榜单热度摘要
- 内容区分成“分类切换区 + Top 3 焦点区 + 热榜流”
- 强化 Top 3 的视觉差异，不让它们淹没在普通列表里
- 允许新增前端增强：Top 3 焦点区、榜单说明、返回顶部/定位辅助

### 3.2 关于本站

定位为“站点名片 + 叙事页”。

- Hero 使用非对称、偏编辑感排版
- 左侧聚焦站点身份与一句话介绍，右侧聚焦作者卡与站点摘要
- 内容区重排为“作者名片 / 站点理念 / 兴趣与关键词 / 联系方式”四段
- 允许新增前端增强：里程碑、当前关注、FAQ 式联系说明
- 页面节奏以阅读体验为先，留白更大、文案块更稳

### 3.3 网站导航

定位为“索引图谱页”。

- Hero 突出站外资源索引性质
- 内容区保持分类分组，但加强章节感、统计感和过滤效率
- 允许新增前端增强：关键词过滤、本地最近访问、分类快速索引
- 不做收藏同步、登录态书签等需要新后端能力的功能

### 3.4 个人中心

定位为“账户工作台”。

- Hero 升级为账户概览区，显示身份、当前激活模块和核心统计
- 主体仍保留左右结构，但增强工作区语义
- 右侧主内容区增加“当前模块头部”，根据 tab 展示不同说明
- 允许新增前端增强：快捷操作区、账户完成度、最近操作摘要（仅基于已加载数据）

## 4. 允许新增的功能边界

仅允许新增前端可闭环的轻功能：

- 本地筛选、局部排序、回到顶部、快速定位
- localStorage 级别的最近访问记录
- 基于现有数据计算出的摘要、完成度、最近动态、统计块

不允许：

- 新接口
- 新业务权限流
- 新持久化字段
- 需要后端支持的新收藏/订阅/多维筛选体系

## 5. 技术边界

- 继续遵守设计系统分层：Tokens -> Foundation -> UI 抽象 -> Business -> Pages
- 业务页不直接引入 `naive-ui`
- 不在业务页写硬编码颜色
- 尽量复用现有 `FeatureCard`、`PageShell`、`S*` 组件与已存在的 section 语义
- 不强行抽“万能 Hero 组件”，以免牺牲页面个性

## 6. 验证策略

先扩展 `scripts/page-ui-contract.test.mjs`，锁定以下结果：

- 四个目标页面都包含新的 hero 或 workspace 结构锚点
- 热门页存在 Top 3 焦点区与榜单工作区
- 关于本站存在叙事型 hero 和重排后的章节化内容
- 网站导航存在过滤或索引增强结构
- 个人中心存在账户 hero 与动态模块头部

完成后再执行：

```bash
node --test scripts/page-ui-contract.test.mjs
npm run style:guard
npm run test:style-guard
npm run build
```
